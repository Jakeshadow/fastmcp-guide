Run DiffusionGemma on LM Studio

LM Studio is the easiest way to run local LLMs — download, click, chat. Here's how to get DiffusionGemma working with full diffusion speed.

Setup Guide Diffusion Config

Install & Set Up LM Studio

Step 1: Download LM Studio

# Download from https://lmstudio.ai
# Available for macOS, Windows, Linux (AppImage)
# Requires GPU with 16GB+ VRAM for DiffusionGemma 26B Q4

Step 2: Load DiffusionGemma

LM Studio 0.3.x supports GGUF models. After downloading a DiffusionGemma GGUF from our GGUF guide, drag the .gguf file into LM Studio's model folder:

# Default model paths:
# Windows: %USERPROFILE%\.cache\lm-studio\models\
# macOS:   ~/.cache/lm-studio/models/
# Linux:   ~/.cache/lm-studio/models/

# Copy your GGUF file there
cp diffusiongemma-26b-q4_k_m.gguf ~/.cache/lm-studio/models/

Step 3: Select Model & Load

Open LM Studio, click the Model Browser (top-left folder icon), find "diffusiongemma" in your local models, and click Load Model. LM Studio auto-detects GPU and loads with llama.cpp backend.

Diffusion-Specific Configuration

GPU Offload

In the right sidebar, under GPU Offload, set layers to max (drag slider all the way right). For a 26B Q4 model, this loads the full model into GPU memory — ~16GB VRAM.

Context Length

Set Context Length to 65536 (DiffusionGemma's native window). Lower to 32768 if you're tight on VRAM.

Diffusion Parameters

In Advanced Settings, add these overrides for diffusion generation:

{
  "diffusion_steps": 8,
  "block_size": 256,
  "temperature": 0.7,
  "top_p": 0.9
}

These parameters control DiffusionGemma's block-level parallel generation. 8 steps is the default; lower is faster but lower quality.

Start the Server

Toggle Local Server in the top bar. LM Studio exposes an OpenAI-compatible endpoint at http://localhost:1234/v1.

curl http://localhost:1234/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "diffusiongemma-26b",
    "messages": [{"role": "user", "content": "Hello!"}],
    "max_tokens": 256
  }'

Troubleshooting

Model doesn't appear in LM Studio browser

LM Studio only detects .gguf files. If your file has a different extension, rename it. Also verify the path — LM Studio scans its model folder on startup.

"Unsupported model architecture" on load

LM Studio bundles its own llama.cpp build. DiffusionGemma support requires llama.cpp with diffusion patches (PR #24427). As of June 2026, not all LM Studio builds include this. You may need to use the standalone llama.cpp setup instead.

OOM (Out of Memory)

LM Studio shows VRAM usage in the bottom bar. If it hits 100%, lower GPU Offload layers or switch to a smaller quant (Q4_0 instead of Q4_K_M). Close other GPU apps (browser, video player) before loading.

Server starts but returns errors

# Test basic connectivity
curl http://localhost:1234/v1/models

# Check LM Studio logs: View → Developer Tools → Console
