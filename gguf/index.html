DiffusionGemma GGUF: Load in Ollama & llama.cpp

GGUF is the standard format for local LLMs — but Ollama and LM Studio can't load DiffusionGemma GGUF yet (blocked on llama.cpp PR #24427). Here's what works, what doesn't, and how to get it running today.

Compatibility Status Download GGUF Convert Yourself

Why Ollama & LM Studio Can't Load GGUF Yet

The diffusion-gemma architecture is brand new (released June 10, 2026). llama.cpp support lives in PR #24427, which hasn't been merged into mainline yet. Ollama and LM Studio both build on mainline llama.cpp — so neither can load DiffusionGemma GGUF until the PR lands.

EngineGGUF SupportWorkaround
llama.cppWorks (PR branch)Build from diffusion-support branch
OllamaBlockedCustom Modelfile + PR branch llama.cpp. See step-by-step
LM StudioBlockedNot possible yet — wait for PR merge
vLLMWorks (native)Doesn't use GGUF — uses HF format directly

If you see "model type diffusion_gemma not supported" or "failed to load gguf model" — you're using mainline llama.cpp, not the PR branch. The error is telling you the architecture isn't recognized. Until PR #24427 merges, the only path is building from the diffusion-support branch. The Ollama setup guide has the exact commands.

Pre-Converted GGUF Downloads

Community-converted GGUF files are available on HuggingFace. Note: These require llama.cpp from the PR branch, not mainline. These are the most common quantization levels:

QuantSizeVRAM NeededQuality
Q4_0~14GB16GBGood — small quality loss
Q4_K_M~15GB16–18GBBest balance — recommended
Q5_K_M~17GB20GBBetter quality, needs more VRAM
Q8_0~26GB28GB+Near-perfect, for 3090/4090

Recommended: Start with Q4_K_M. It fits on most 24GB GPUs and offers the best quality-to-size ratio. If you have a 16GB GPU, use Q4_0.

Convert from PyTorch to GGUF

Step 1: Download the Original Model

# From HuggingFace
huggingface-cli download google/diffusiongemma-26b-A4B-it \
  --local-dir ./diffusiongemma-26b

# Or use git-lfs
git lfs install
git clone https://huggingface.co/google/diffusiongemma-26b-A4B-it

Step 2: Convert to FP16 GGUF

cd llama.cpp
python convert_hf_to_gguf.py \
  ../diffusiongemma-26b \
  --outtype f16 \
  --outfile diffusiongemma-26b-f16.gguf

Step 3: Quantize to 4-bit

./build/bin/llama-quantize \
  diffusiongemma-26b-f16.gguf \
  diffusiongemma-26b-q4_k_m.gguf \
  Q4_K_M

Step 4: Verify the GGUF

./build/bin/llama-cli \
  -m diffusiongemma-26b-q4_k_m.gguf \
  -p "Hello world" \
  -n 32

GGUF FAQ

Why does Ollama say "model type diffusion_gemma not supported"?
Ollama builds on mainline llama.cpp, which doesn't have DiffusionGemma support yet. The diffusion-gemma architecture is only in PR #24427. Until it's merged into mainline, Ollama can't recognize DiffusionGemma GGUF files. Our Ollama guide has a workaround using a custom Modelfile.
Why does LM Studio fail with "model type not supported"?
Same issue as Ollama — LM Studio uses mainline llama.cpp for both MLX and Metal backends. The diffusion-gemma architecture isn't in mainline. There's no workaround for LM Studio yet; you'll need to wait for PR #24427 to merge, then LM Studio to update their llama.cpp dependency.
Can I use the pre-converted GGUF files today?
Yes, but only with llama.cpp built from the diffusion-support branch (PR #24427). The unsloth GGUF on HuggingFace has 107k+ downloads — all using this branch. Once the PR merges, these GGUF files will work with mainline llama.cpp, Ollama, and LM Studio.
Which GGUF quantization should I choose?
Q4_K_M is the recommended starting point — it fits on most 24 GB GPUs with the best quality-to-size ratio. Q4_0 for 16 GB cards. Q5_K_M for better quality if you have 20+ GB VRAM. Q8_0 if VRAM isn't a constraint (28+ GB).
Does vLLM use GGUF format?
No. vLLM loads models directly from HuggingFace format (PyTorch/safetensors), not GGUF. If you're using vLLM, you don't need GGUF at all. vLLM has supported DiffusionGemma since day 0 with Google's collaboration.
